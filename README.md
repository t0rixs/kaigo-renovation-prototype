# 介護住宅改修 図面・見積アプリ

介護住宅改修向けの図面作成・簡易見積を、Android、iPhone、ブラウザで使えるFlutterアプリにしたものです。

## 主な機能

- 基本情報の入力と端末内への自動保存
- 250mm補助線・500mm主線の方眼図面と、mm単位の内部座標
- ドラッグによる間取り作成、移動、右下の丸ハンドルによるサイズ変更
- 間取り・設備の名称を場所名へ統一し、間取り内へ置いた手すり・設備へ場所名を自動設定
- 縦横に固定した手すりの作図、移動、両端ハンドルによる長さ・縦横方向の変更と施工属性
- 作成ドラッグは未選択要素の当たり判定より優先し、間取りの縁と全区間重なる手すりだけ配置を抑止
- 手すり商品の一覧・追加・編集・削除と、屋内・屋外デフォルト商品の設定
- 商品ごとの対応環境、直径、手すり・ジョイント・柱単価、最大ジョイント間隔の管理
- 手すり両端と商品ごとの最大間隔に応じたジョイント自動配置・見積反映
- 屋内・屋外に対応した商品絞り込みと、手すりごとの商品変更
- 壁取付型・独立型とも任意位置へ配置し、独立型はジョイント数と同数の柱を自動計上
- 2本の手すりの端点を接続すると、編集単位を保ったままL字手すりとして見積を統合
- 最大ジョイント間隔を超えない範囲での各区間の均等割り配置
- トイレ設備記号のタップ配置、移動、サイズ変更
- トイレ設備記号の0°・90°・180°・270°回転
- 間取り境界へスナップするドア・窓の配置、壁上移動、幅変更
- ドアツールから開き戸・スライド戸を選択し、図面記号を描き分け
- ドア・窓を選択後の通常ドラッグで別の壁へ付け替え
- ドアを選択後の通常ドラッグで壁の内外へまたぐ内開き・外開き切替
- 開き戸の左右反転とワンタップ内開き・外開き切替、スライド戸の引き方向反転
- 間取り内部は配置操作を妨げず、四辺だけを選択・移動判定に使用
- どの作図モードでも既存要素をタップで選択し、通常ドラッグで移動、丸ハンドルでリサイズ
- リサイズ中も指位置に最も近い250mmグリッドへ追従
- 削除、元に戻す、やり直し
- 図面上の全手すりについて、本体・ジョイント・柱の材料原価を自動計算
- 全作図モードでのピンチズーム、スマホ・タブレット・PC対応
- 選択中のツールを再タップして解除し、通常ドラッグで画面移動
- 図面の空白をワンタップして選択解除
- 基本情報、図面、見積内容の端末内自動保存

## MVPのデータ保存

商品マスターと全案件を、`schemaVersion: 1`の単一JSON文書として保存します。各案件には基本情報、図面オブジェクト、手すり、再計算済みの見積スナップショットが含まれます。

```text
productMaster
projects[]
  basicInfo
  drawing
    objects
    handrails
  estimate
activeProjectId
```

- Android / iOS: アプリのDocuments領域に`kaigo_renovation_mvp_v1.json`を保存
- Web: 同じJSON文書をブラウザのローカル領域へ保存
- 旧データ: 初回起動時のみ従来のSharedPreferencesデータを読み込み、新形式へ移行

読み書きは[`AppDataRepository`](lib/storage/app_data_repository_base.dart)に分離しています。サーバー化するときは、この実装をHTTP API用リポジトリへ差し替え、JSONスキーマをそのままリクエスト・レスポンスに利用できます。

## 起動

```bash
flutter pub get
flutter run
```

接続先を指定する場合:

```bash
flutter devices
flutter run -d <device-id>
```

## テストとビルド

```bash
flutter analyze
flutter test
flutter build apk --debug
flutter build ios --simulator --debug
./tool/build_web.sh
```

Webビルドでは、以前のPWA版が端末に残したキャッシュを削除する退役用Service Workerも同時に配置します。

## GitHub Pages

`main`へpushすると、GitHub Actionsが解析・テスト・Flutter Webビルドを行い、GitHub Pagesへ自動公開します。リポジトリ名に応じた`base-href`もワークフロー内で設定されます。

ストア公開時は、Androidの`applicationId`、iOSのBundle Identifier、署名設定、アプリアイコンを本番用に設定してください。
